post /item/add
(postItemAdd)
POST a new item. Requires user login. Also requires fields name, description, price, location, delivery and category. Supports images, but those are not required. Successfully created entry is added to the item database/array.

Item - Item Up

Stores data for items. Some data is grabbed from User.
itemID (optional)
UUID Unique item ID, generated randomly format: uuid
name
String Item name
description
String Item description
price
String Price the seller has set for this product
location
String Location of item; ex. New York or Helsinki, Finland
category
String Item category. Examples: clothing, vehicle
image
array[Object] Item images
deliveryMethod
String Accepted delivery methods, ex. shipping or pickup
datetime (optional)
Date Date and time the item was created on. format: date-time
ownerID (optional)
String Unique ownerID, grabbed when creating listing.
ownerName (optional)
String Item owner name (first + last), set automatically when creating listing (while logged in.)
ownerMobileNumber (optional)
BigDecimal Item owner mobile number, set automatically when creating listing (while logged in.)
ownerEmail (optional)
String Item owner email, set automatically when creating listing (while logged in.) format: email

User - User Up

Stores all data for user(s).
userID (optional)
String Unique identifier for the given user. format: string
example: 12h844u3f20
fName
String First name of user
example: Alice
lName
String Last name of user
example: Smith
email
String User email format: email
mobileNumber
BigDecimal User's mobile/phone number
example: 112355346
username
String Account username, used with password to log in.
example: alice213
password
String User password, encrypted when added to the database/array format: password
example: example
createdOn (optional)
String The date that the user was created. format: string
example: 1. October 2021 15:32
